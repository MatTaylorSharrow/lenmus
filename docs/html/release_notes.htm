LenMus Phonascus 3.4 Release Notes

What's new in 3.4

Release 3.4 includes, mainly, technical changes an some bug fixes:

  • A new eMusicBooks source format, based on XML and DocBook, has been developed with three objectives in mind:
    • To simplify writing eMusicBooks.
    • To simplify the translation of eMusicBooks.
    • To enable new features.
  • The controller to display eMusicBooks has been rewritten to adapt to the new eMusicBooks format. Now the index panel can also display images.
  • A program, named LangTool, has been developed to compile eMusicBooks and to automate the translation process.
  • Supporting wxWidgets libraries upgraded to version 2.8.0
  • Several bug fixes and other technical improvements.

[1610190] In the options panel, key signature 'D mayor/B minor' is duplicated and 'D b Mayor/B b minor' is missing.

[1604535 ] Minor scales, in scales exercises, have wrong key signature
When the option 'draw key signature' is enabled, minor scales are drawn with the major scale key signature and the necessary accidentals added to the notes. The minor scale key signature should be used instead. Now fixed.
[1601350 ] No ascending intervals in aural training
In exercise for aural training, intervals identification, when you select the tree types (ascending descending and harmonic) most of the generated intervals are harmonic and a few descending, but never ascending. Now fixed.
[1610186] Spanish translation error in aural training exercises.
Label 'identify scales' was translated as 'identify intervals'. Now fixed.
Metronome lose previous run settings. Always initialised at 20.
Setting the spin ctrol triggers a EVT_SPINCTRL that causes a metronome reset. As this worked ok in previous versions I assume it is due to a change in wxSpinCtrol behaviour in new wxWidgets release. Solved by changing metronome initialization code.
