LenMus package preparation check list

  1. Prepare executables and books
    1. Change version string in TheApp::GetVersionNumber(). No need to change release flags in TheApp.cpp as they are changed automatically by pre-processor directives when choosing Release build.
    2. Create the executable
    3. Pack the help file
    4. Pack the eBooks
    5. Review the translations (.po) files

  2. Create package
    1. Change version string in installer (installer.nsi)
    2. Generate and test the installer

  3. Test package
    1. Install package in another machine
    2. Test it

  4. Update local web site
    1. Update 'installation.htm' (in 'docs/htm').
    2. downloads.htm (en, sp)
    3. new_in_version.htm (en, sp)
    4. features.htm (en, sp)
    5. release_notes_xx.htm (en)
    6. screenshots.htm (en, sp)

  5. Prepare release
    1. Update 'UpdateData.xml' (in 'docs/src'). Add size of package.
    2. SVN: tag release as 'Release x.x nnnn x'
    3. Prepare release description for SourceForge
    4. Prepare news item for web lenmus.org and for SourceForge

  6. Upload package
    1. FTP (in clear, no SSH secured): upload to "upload.sourceforge.net" into the "/incoming" directory
    2. Logon to Sourceforge website
    3. Go to the "lenmus file release" page (Admin / File Releases)
    4. At the botton, click on "Add release"
    5. Fill in information about the release (Name: 'LenMus x.x. Installer for Windows') You should see a list of all the files uploaded today. Tick the checkbox for the lenmus package file and click "Add files" button.
    6. Go to the downloads page and check that you can download the new release.
    7. Upload 'UpdateData.xml' to the web server

  7. Send news in sourceforge
    1. Go to the LenMus News page (Summary / News)
    2. After the menu press 'Submit'. Fill in news information.
    3. Send sourceforge notice to file release tracker

  8. Update website
    1. News
    2. Downloads page
    3. Release notes page
    4. Other
